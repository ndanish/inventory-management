Key Design Decisions

Pattern
Reactivity Model
Raw data stored in refs. Derived views as computed properties. Filters watched to re-trigger API calls — no client-side re-filtering.
Pattern
No Database
All JSON loaded into Python memory at startup via mock_data.py. Filtering is in-process list comprehension. Purchase orders written back to JSON at runtime.
Pattern
Shared Filter State
useFilters() composable is a singleton — all views share the same filter refs, so changing a filter on one page persists when navigating to another.
